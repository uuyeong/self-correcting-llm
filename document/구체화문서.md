# 파이널 프로젝트 구체화 문서

## 📌 프로젝트 개요

### 제목

**Self-Correcting LLM: Hallucination Detection and Reliability-Controlled Re-generation via Internal Activation Analysis**

### 한 줄 요약

LLM이 텍스트를 생성하는 과정에서 내부 활성화(Hidden States)를 실시간 분석하여 할루시네이션을 탐지하고, 탐지 신호의 강도에 따라 재생성 전략을 다르게 적용하는 신뢰 기반 생성 파이프라인을 제안한다.

### 기존 연구와의 차별점

| 기존 연구 | 이 프로젝트 |
| --- | --- |
| 탐지(Detection)만 수행 | 탐지 → 재생성까지 통합 파이프라인 |
| 단일 재생성 전략 | 신호 강도에 따라 3단계 재생성 전략 차등 적용 |
| 단일 모델/도메인 | 모델 크기별(3B vs 8B), 도메인별 비교 실험 |
| 영어 LLM 위주 | 도메인별(의학/법률/일반) 표상 분석 포함 |

---

## 🧠 핵심 아이디어

### 문제 정의

LLM은 사실과 다른 정보를 자신감 있게 생성하는 "할루시네이션" 문제를 가지고 있다. 기존 탐지 연구는 탐지에서 멈추지만, 탐지 이후 어떻게 생성을 수정할지는 미해결 문제다.

### 제안하는 파이프라인

```
[입력 질문]
    ↓
[LLM 생성 시작]
    ↓
[Hidden States 실시간 추출]
    ↓
[Probing Classifier → 할루시네이션 신호 강도 측정]
    ↓
┌──────────────────────────────────────────────────┐
│  신호 강도에 따른 3단계 전략                      │
│  LOW  → 그대로 출력 (추가 비용 없음)              │
│  MID  → 부분 보정 (Logit 대조 or Top-p 조정)     │
│  HIGH → 완전 재생성 (Re-generation)              │
└──────────────────────────────────────────────────┘
    ↓
[최종 출력 + 신뢰도 점수 표시]
```

### 핵심 가설

- **가설 1**: LLM의 특정 레이어 Hidden States는 할루시네이션 발생 전에 이미 이상 패턴을 보인다
- **가설 2**: 탐지 신호 강도에 따라 재생성 전략을 차등 적용하면, 단일 전략(항상 완전 재생성) 대비 사실 정확도는 유지하면서 Token Cost와 Latency를 유의미하게 줄일 수 있다
- **가설 3**: 모델 크기가 클수록 Hidden States의 할루시네이션 신호가 더 명확하게 분리된다
- **가설 4**: 도메인(의학/법률/일반상식)에 따라 할루시네이션 정보가 인코딩되는 최적 레이어 위치가 다르다

## 🔬 실험 설계 (다각도 평가)

### 실험 1: Probing Classifier 레이어별 성능 분석

- LLaMA-3.1 8B의 레이어 각각에서 hidden states 추출
- 레이어별 할루시네이션 탐지 정확도(AUROC) 측정
- 시각화: 레이어별 탐지 정확도 히트맵

### 실험 2: 3단계 재생성 전략 효과 비교

- LOW/MID/HIGH 신호별 전략 적용 후 사실 정확도 비교
- 측정 지표:
    - Factual Accuracy: Baseline 대비 향상 수치 (%)
    - Token Cost: 전략별 평균 추가 토큰 수
    - Latency: 전략별 평균 응답 시간 (ms)
    - **효율 지표 직접 설계**: `Efficiency = Accuracy Gain / (Token Cost × Latency)`
- 시각화: 정확도 / 토큰 비용 / 지연시간 3축 비교 그래프

### 실험 3: MID 전략 내부 비교 (전략 A vs B)

- 전략 A: Top-p 축소 (0.9 → 0.7), 불확실한 토큰 후보를 줄여 보수적 생성
- 전략 B: DoLa 스타일 Logit 대조 (상위 레이어 logit - 하위 레이어 logit), 사실성 방향으로 디코딩 편향
- 어떤 MID 전략이 정확도/비용 면에서 더 효과적인지 비교
- 시각화: A vs B 정확도 및 latency 비교 표

### 실험 4: 모델 크기별 비교 (3B vs 8B)

> **변경**: LLaMA-2 7B/13B → LLaMA-3.2 3B / LLaMA-3.1 8B (실험 취지 동일)

- 동일 실험을 3B와 8B에서 반복
- Hidden States 분리도(Linear Separability) 비교
- 시각화: t-SNE로 참/거짓 Hidden States 분포 비교

### 실험 5: 도메인별 최적 레이어 분석

- 의학/법률/일반상식 도메인으로 데이터 분리 (PopQA 기반)
- 목표: 단순히 "최적 레이어 찾기"가 아닌 **"도메인별 정보 처리 방식의 차이 이해"**
- 분석 프레임:
    - 탐지 정확도가 높은 도메인 (일반상식) → 해당 레이어에서 사실성 정보가 명확히 인코딩됨
    - 탐지 정확도가 낮은 도메인 (의학/법률) → "전문 도메인 지식은 일반 사실성과 다른 레이어에서 처리된다"는 새로운 발견으로 해석
    - 도메인별 최적 레이어 위치(얕은 레이어 vs 깊은 레이어)의 차이가 의미하는 바 논의
- 시각화: 도메인별 레이어 중요도 히트맵

보고서 서술 방향 예시:

> "의학 도메인에서 탐지 정확도가 X% 낮게 나타났다. 이는 단순 실패가 아니라, 전문 도메인 지식이 모델 내부에서 일반 사실성과 다른 레이어에서 표상된다는 것을 시사한다. 이는 향후 도메인 특화 Probing Classifier의 필요성을 보여주는 근거가 된다."
> 

### 실험 6: Baseline 종합 비교

- Baseline 1: 탐지 없이 그냥 생성
- Baseline 2: 항상 완전 재생성 (단일 전략)
- Ours: 신호 강도 기반 3단계 전략
- 시각화: 사실 정확도 / 토큰 비용 / 지연시간 종합 비교 표
